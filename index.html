Elite - Per-Listing Price

The Elite price is based on the value a listing generates for the agent. We take the median property value in a district, estimate the commission an agent would earn on a deal, factor in how many leads each listing receives and the likelihood of conversion, then apply our take rate.

BigQuery View - datasets.price_calculator
price_per_listing = median_lead_value x 3% (commission) x 3% (conversion) x leads_per_listing (3 months) x 6% (take rate)

Uses a 12-month rolling window. Districts need 67+ avg monthly leads and top 10 per country to qualify, otherwise bucketed as "Other". Leads per listing is the average over 3 months, calculated across all countries using only Elite and Elite Plus listings. The resulting price_per_listing is a 3-month rate.

Elite Period Pricing

The per-listing price is a 3-month rate. To get the total, we multiply it by the number of listings and the number of 3-month chunks in the period (1 for 3m, 2 for 6m, 4 for 12m). Longer commitments get a discount: 10% off for 6 months, 20% off for 12 months.

Elite - Listing cost per period
3 months: price_per_listing x listings
6 months: price_per_listing x 2 x 0.9 x listings (10% off)
12 months: price_per_listing x 4 x 0.8 x listings (20% off)

Elite Plus - How It Differs

Elite Plus = Elite listing cost + ELP (Elite Listing Promotion) cost. The per-listing price uses a 7% take rate instead of Elite's 6%. Each tier includes ELPs, and users can add 0-10 more on top.

Demand Multiplier
demand_multiplier = clamp(current_elp_count / ideal_slots, 0.9, 1.3)

Ideal slots = district impressions / 400,000. Fewer ELPs than ideal means multiplier drops toward 0.9 (lower price). Saturated means it pushes toward 1.3 (higher price). Applied to ELP pricing only.

ELP cost per period
ELP 3m: price_elp x 3 x total_elps
ELP 6m: price_elp x 3 x total_elps x 2
ELP 12m: price_elp x 3 x total_elps x 4
price_elp is a monthly value, x3 gives the quarterly price
